复杂度分析

  • 复杂度也叫渐进复杂度，包括时间复杂度和空间复杂度，用来分析算法执行效率与数据规模之间的增长关系，可以粗略地表示，越高阶复杂度的算法，执行效率越低
数据结构的意义：
  • 让代码执行更快
  • 让代码更省存储空间
数据结构考量指标
  • 执行效率
执行效率指标
  • 时间复杂度
  • 空间复杂度
类别
  • 最好情况时间复杂度（best case time complexity）在最理想的情况下，执行这段代码的时间复杂度
  • 最坏情况时间复杂度（worst case time complexity）在最糟糕的情况下，执行这段代码的时间复杂度
  • 平均情况时间复杂度（average case time complexity）
  • 均摊时间复杂度（amortized time complexity）

数组

  • 一种线性表数据结构,用一串连续的内存空间,来存储一组具有相同类型的数据
线性表
  • 每个表上的数据最多只有前和后两个方向
  • 数组、链表、队列、栈…都是线性表结构
  • 与线性表结构对应的是 非线性表 :二叉树、堆、图…
连续的内存空间和相同类型的数据
利弊

  • 利：因为以上两个限制，所以线性表数据结构的特性是 ：随机访问。简单的说：数组支持随机访问，根据下标随机访问的时间复杂度为 O(1）,平均情况时间复杂度为 O(n)

  • 弊：删除、插入操作，因为保持连续性需要做大量的数据搬迁工作

链表

  • 通过“指针”将一组零散的内存块串联起来使用,把内存块称为链表的“结点”。为了将所有的结点串起来，每个链表的结点除了存储数据之外，还需要记录链上的下一个结点的地址,这个记录下个结点地址的指针叫作后继指针 next
  • 第一个结点叫作头结点，把最后一个结点叫作尾结点。其中，头结点用来记录链表的基地址。有了它，我们就可以遍历得到整条链表。而尾结点特殊的地方是：指针不是指向下一个结点，而是指向一个空地址 NULL，表示这是链表上最后一个结点
链表类别
  • 单链表 尾节点指向NULL，是最后的结点
  • 双向链表 (特殊的单链表)，尾结点指针指向链表的头结点。当处理的数据具有环形结构特点，从链尾到链头很方便
  • 循环链表 支持两个方向，每个结点不止有一个后继指针 next 指向后面的结点，还有一个前驱指针 prev 指向前面的结点
利弊
  • 利：因为不追求连贯性，对于链表而言，删除、插入是十分快速。只需要考虑相邻节点的指针该表，所以时间复杂度是 O(1)。支持动态扩容
  • 弊：显而易见，因为内存的不连贯性，所以无法直接的根据坐标获取对应值，只能做遍历操作。访问性能需要 O(n) 的时间复杂度
设计思想
  • 对于执行较慢的程序，可以通过消耗更多的内存（空间换时间）来进行优化，比如 缓存；而消耗过多内存的程序，可以通过消耗更多的时间（时间换空间）来降低内存的消耗
  • 当对于内存苛刻的时候，数组比较适合。链表的每个结点需要消耗额外的存储空间去存储一份指向下一个结点的指针，内存消耗会翻倍，链表的频繁删除与插入操作会导致频繁的内存申请和释放，容易导致内存碎片。在Java中，该操作会导致频繁的GC
链表技巧
  • 理解指针
    • 将某个变量赋值给指针，实际上就是将这个变量的地址赋值给指针，或者反过来说，指针中存储了这个变量的内存地址，指向了这个变量，通过指针就能找到这个变量
  • 明确指针指向与内存泄露
    • 尤其对于C，删除链表结点时，一定要记得手动释放内存空间
  • 妥善使用 哨兵结点

栈

  • 后进者先出，先进者后出，这就是典型的栈结构，只允许一端插入和删除数据，所以只需要一个栈顶指针，是一种操作受限的线性表
操作
  • 出栈 与入栈
区分
  • 数组实现的栈叫做顺序栈，而链表实现的栈叫做链式栈

不管是顺序栈还是链式栈，入栈、出栈只涉及栈顶个别数据的操作，所以时间复杂度都是 O(1)

队列

  • 先进先出，但是不同于栈，队列需要两个指针：一个是 head 指针，指向队头；一个是 tail 指针，指向队尾也是一种操作受限的线性表数据结构
操作
  • 入队 enqueue()，放一个数据到队列尾部；出队 dequeue()，从队列头部取一个元素
区分
  • 用数组实现的队列叫作顺序队列，用链表实现的队列叫作链式队列

递归

  • 递归本身就是根据栈实现的，在使用过程中会存在很多弊端（堆栈溢出、重复计算、函数调用耗时多、空间复杂度高…）
