使用本地图片

误区：
  • 最开始以为是需要使用网页链接。因为晚上的推荐基本说是使用七牛云呀或者git做图片存储。所以一度都是先在简书上粘贴图片，生成链接，再复制在md编辑器。
新方式

  • npm install hexo-asset-image –save

  • npm因为和谐的问题，所以建议使用cnpm或者更换淘宝镜像

  • hexo根目下的_config.yml设置

    post_asset_folder: true

  • 当 hexo new "xx"时候生成新的文章时候，会在对应位置生成对应文件夹

  • 但是在实际使用时候发现，导入进去还是有问题。本地是显示了，但是提交上去之后。啥都没有，但是别人也是这样的，让我很绝望！偶然发现网上别人的一个方法：

  • 在Typora中, 选择Edit > Image Tools > Use Image Root Path， 然后选择你要的相关文件夹，选择之后，在md文件头上 会多一个 typora-root-url: xxxx, 好了，现在可以正常显示了。

  • 最后,格式如下，提交就可以显示了 不要带 / 符号
    ![](怎么回事小老弟.gif)

标签云

  • npm install hexo-tag-cloud@^2.0.* –save

  • 对应的主题文件夹下 weiget找到 tag的配置文件

warning: LF will be replaced by CRLF in ……

  • git config –global core.autocrlf false
    • 原因是路径中存在 / 的符号转义问题，false就是不转换符号默认是true，相当于把路径的 / 符号进行转义，这样添加的时候就有问题
