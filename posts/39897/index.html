完整第三方资料

  • 基于此资料做的笔记，看不到其中文档的图片示例的话，那就只能小✈全开

基础

  • 访问控制只有公开和私有两种
  • 所有首字母大写的类型、结构体、字段、变量、函数、方法等都是公开的，可供其他包使用。而小写的则是私有的，只能在包内使用。
函数调用

函数是通过<pkgName>.<funcName>的方式调用的，这一点和Python十分相似。包名和包所在的文件夹名可以是不同的，此处的<pkgName>即为通过package <pkgName>声明的包名，而非文件夹名。

变量定义
  • :=这个符号直接取代了var和type,这种形式叫做简短声明。不过它有一个限制，那就是它只能用在函数内部；在函数外部使用则会无法编译通过，所以一般用var方式来定义全局变量
  • _（下划线）是个特殊的变量名，任何赋予它的值都会被丢弃
访问修饰符的定义
  • 大写字母开头的变量是可导出的，也就是其它包可以读取的，是公有变量；小写字母开头的就是不可导出的，是私有变量，只能在包内使用
  • 大写字母开头的函数也是一样，相当于class中的带public关键词的公有函数；小写字母开头的就是有private关键词的私有函数
数组与切片
  • 数组是一种 值类型（不像 C/C++ 中是指向首元素的指针），所以也可以通过 new() 来创建：
    • var arr1 = new([5]int)
    • 那么这种方式和 var arr2 [5]int 的区别是什么呢？arr1 的类型是 *[5]int，而 arr2的类型是 [5]int。这样的结果就是当把一个数组赋值给另一个时，需要再做一次数组内存的拷贝操作
  • 切片是一种引用类型
    • 声明切片的格式是： var identifier []type（不需要说明长度）一个切片在未初始化之前默认为 nil，长度为 0
    • 切片的初始化格式是：var slice1 []type = arr1[start:end]。
  • slice 可理解为动态数组，即初始定义时候不明确长度，但是实际是一个引用类型。slice总是指向一个底层array
  • map 常规字典
实例创建

都是用于内存分配

  • *make(T) *返回一个类型为 T 的初始值，它只适用于3种内建的引用类型：slice、map 和 channel
  • new(T) *为每个新的类型T分配一片内存，初始化为 0 并且返回类型为T的内存地址：这种方法 **返回一个指向类型为 T，值为 0 的地址的指针，它适用于值类型如数组和结构体）；它相当于 &T{}，且需要显示返回指针情况，不常用
  • new 函数分配内存，make 函数初始化

函数

传值与传指针

  • channel，slice，map这三种类型的实现机制类似指针，所以可以直接传递，而不用取地址后传递指针。（注：若函数需改变slice的长度，则仍需要取地址传递指针）

  • 常规操作时候，需要操作某个变量的值。使用&v将实际地址传向参数类型为*T的函数中

defer
  • 延迟语句，函数执行到最后的时候会按照逆序执行这些个defer修饰的操作。后进先出，最先写的方法最后才会被调用
  • 常用来做关闭资源等操作
保留函数
  • Go里面有两个保留的函数：init函数（能够应用于所有的package）和main函数（只能应用于package main）。这两个函数在定义时不能有任何的参数和返回值
  • 如果 package main中同时存在main和init函数，那么会先执行init再执行main

方法

  • 与函数不同的是，方法是一种带有指定接收者的函数
  • 虽然method的名字一模一样，但是如果接收者不一样，那么method就不一样。即：只有对应的接收者才能够调用该函数
  • method里面可以访问接收者的字段
  • 调用method通过.访问，就像struct里面访问字段一样

interface 🛫

  • 是一组方法组合，我们通过interface来定义对象的一组行为

  • interface就是一组抽象方法的集合，它必须由其他非interface类型实现，而不能自我实现

接口函数参数
  • interface的变量可以持有任意实现该interface类型的对象，通过定义interface参数，让函数接受各种类型的参数
接口实现

  • 给实现者定义方法，方法与接口里面的签名（名称、参数、返回值）一样，就是实现了接口

  • 接口的实现者必须是一个具体的类型，且必须实现了接口所有的方法才算是实现了接口

并发

goroutine

  • goroutine是通过Go的runtime管理的一个线程管理器。goroutine通过go关键字实现了，其实就是一个普通的函数

  • 说白了就是协程，运行在相同的地址空间、内存共享的，但是不要通过共享来通信，而要通过通信来共享

channel
  • 提供给goroutine作为数据通信的一种通信机制
  • 非缓存类型，容量是0，不能存任何数据，只起传输作用，不做数据停留
  • 使用内置的make直接声明即可，内置 close函数关闭
  • chan只有两个操作（发送和接收）

Buffered Channel

  • 缓冲型Channel，类似可阻塞队列，允许指定缓冲大小，先进先出
  • 通过make的函数的第二个函数即可指定容

单向 channel

range
  • 用于读取slice、map、channel数据
select
  • 用于选择不同类型的通讯，监听Channel
  • 设置超时

Sync

sync.Mutex
sync.Once
  • 用以让代码只执行一次
sync.WaitGroup
  • 确保协程执行完毕后，及时退出
sync.RWMutex
  • 解决资源竞争，提高性能

Context

  • 作为一个并发安全的接口，具备手动、定时、超时发出取消信号、传值等功能，主要用于控制多个协程之间的协作；尤其是取消操作

知识点

变量逃逸分析
  • 栈用于内存分配，栈的分配和回收速度非常快
  • 堆分配内存和栈分配内存相比，堆适合不可预知大小的内存分配，但是为此付出的代价是分配速度较慢，而且会形成内存碎片
  • 通过编译器分析代码特征何代码的生命周期，决定应该使用堆还是栈来进行内存分配
  • 编译器觉得应该分配在堆和栈的原则是：
    • 变量是否被取地址
    • 变量是否发生逃逸
sync.Map
  • map在并发情况下，只读是安全的，同时读写是线程不安全的。一般做法是加锁，但是性能不高。
  • 1.9版本添加了sync包下的 Map，使用方法与常规的Map不一样，且没有提供获取map数量的方法.替代方法是在获取 sync.Map时遍历自行提取
  • sync.Map为了保证并发安全会有一些性能损失。因此在非并发的情况下，推荐使用常规map
