区别

  • JNI动态库和非JNI动态库的区别是：JNI动态库定义了一个名为 JNI_OnLoader函数，在动态库加载之后会被系统调用，完成JNI函数的注册

静态注册

原理　
  • 根据函数名来找 JNI 函数，Android Studio支持
参考　
  • JNI入门
弊端
  • 后期类名、文件名改动，头文件所有函数将失效，需要手动改，超级麻烦易出错
  • 代码编写不方便，由于JNI层函数的名字必须遵循特定的格式，且名字特别长
  • 会导致程序员的工作量很大，因为必须为所有声明了native函数的java类编写JNI头文件
  • 程序运行效率低，因为初次调用native函数时需要根据根据函数名在JNI层中搜索对应的本地函数，然后建立对应关系，这个过程比较耗时

动态注册

原理
  • JNI 允许我们提供一个函数映射表，注册给 JVM，这样 JVM 就可以用函数映射表来调用相应的函数
    • 因为Java native 函数和 JNI 函数 是一一对应的，所以hi有一种结构来保存着各种关系
实现过程
  • 利用结构体JNINativeMethod保存Java Native函数和JNI函数的对应关系
  • 在一个JNINativeMethod数组中保存所有native函数和JNI函数的对应关系
  • 在Java中通过System.loadLibrary加载完JNI动态库之后，调用JNI_OnLoad函数，开始动态注册
  • JNI_OnLoad中会调用AndroidRuntime::registerNativeMethods函数进行函数注册
  • AndroidRuntime::registerNativeMethods中最终调用jniRegisterNativeMethods完成注册
示例

  • android_media_MediaScanner.cpp

  • AndroidRuntime.cpp

  • JNIHelp.cpp

动态注册的函数调用

  • 当 Java 层通过 System.loadLibrary 加载完 JNI 动态库之后，紧接着会查找库中一个叫做 JNI_OnLoad 的函数

  • 如果有，就调用，而动态注册的工作就是在这里完成

  • android_media_MediaPlayer.cpp

  • JNI 代码层一般都要包含 jni.h 。Android源码中提供了一个帮助头文件JNIHelp.h，内部就包含了jni.h。

弊端
  • 需要编译所有声明了native方法的Java类，每个所生成的class文件 都得用javah生成一个头文件
  • javah生成的JNI层函数名特别长，书写起来很不方便
  • 初次调用native函数时要根据函数名字搜索对应用JNI层函数来建立关联关系，这样会影响运行效率。

数据类型转换

  • Java 调用 native 函数传递的参数是 Java 数据类型，JNI 也是区别对待这二者
    • Java 数据类型也分为 基本数据类型 和 引用数据类型
  • 除了 Java 基本数据类型的数组、Class、String和Throwable之外，其余所有Java对象的数据都在JNI中都用 object 表示
示例

  • MediaScanner.java

  • android_media_MediaScanner.cpp

JNIEnv

  • 一个与线程相关的代表 JNI 环境的结构体
  • 都是native 函数转换成 JNI 函数后由虚拟机传进来
作用
  • 调用 java 函数
  • 操作jobject 对象等很多事情
使用 JNIEnv

  • android_media_MediaPlayer.cpp

  • Java 与 JNIEnv 的关系

    • 调用 JavaVM 的 AttachCurrentThread 函数，可以得到这个线程的 JNIEnv 结构体，从而在后台线程中调用 Java 函数
    • 在线程推出执前，需要调用 JavaVM 的 DetachCuurrentThread 函数来释放对应的资源
JNIEnv 操作 jobject

  • jobject 即 object , 即 对象

  • 对象 由 对象的 成员变量 和 成员函数 组成

  • 成员变量 和 成员函数 都是由 类 来定义

  • 然: 在 JNI 规则中， jfieldID 和 jmethodID 表示 Java的 成员变量和成员函数，JNIEnv 提供了方法

    • jfieldID GetFieldID (jclass clazz ,const char *name, const char *sign);
    • jmethodID GetMethodID (jclass clazz, const char *name,const char *sig);
